# 1. Analysis (10 marks)

This section documents the analysis phase of the project, worth 10 marks in the OCR assessment.

## Contents

- `problem_statement.md` - Clear definition of the problem to be solved
- `stakeholders.md` - Identification and analysis of stakeholders
- `success_criteria.md` - Definition of success criteria
- `requirements_analysis.md` - Detailed requirements analysis
- `constraints.md` - Constraints and limitations

## OCR Assessment Criteria

To achieve full marks in this section, the documentation should demonstrate:

1. **Clear problem definition** - Well-defined problem with context
2. **Stakeholder identification** - All relevant stakeholders identified
3. **Success criteria** - Measurable and appropriate success criteria
4. **Requirements** - Comprehensive functional and non-functional requirements
5. **Constraints** - Realistic constraints identified

## Template Files

Each file in this directory contains a template to guide the analysis documentation. Fill in the details based on your project requirements.
