# Stakeholder Analysis

## 1. Primary Stakeholders

### 1.1 Anthony Lennon (Student)
- **Role**: Student developer, learning software engineering
- **Interest**: Learning, achieving good marks, enjoying the project
- **Influence**: High (active development participant)
- **Requirements**: 
  - Clear learning objectives
  - Manageable complexity
  - Support and guidance
  - Real-world experience

### 1.2 Alex Lennon (Software Lead)
- **Role**: Software lead, mentor
- **Interest**: Project success, Anthony's learning, product quality
- **Influence**: Very High (technical decisions, guidance)
- **Requirements**:
  - Well-structured project
  - Educational value
  - Business value
  - Quality software

### 1.3 Michael Hull (Hardware Lead)
- **Role**: Hardware design lead
- **Interest**: Hardware design, product success
- **Influence**: High (hardware specifications)
- **Requirements**:

  - Software integration support
  - Timely delivery

- **Interest**: Hardware development, collaboration
- **Influence**: Medium (hardware implementation)
- **Requirements**:
  - Clear specifications
  - Collaboration opportunities

### 1.5 OCR Examiners
- **Role**: Assessment evaluators
- **Interest**: Evaluating against assessment criteria
- **Influence**: Very High (marks awarded)
- **Requirements**:
